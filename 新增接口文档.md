# 新增接口文档

本文档记录从 `C:\develop\campus-frontend` 迁移前端时发现的新增前端接口需求。当前实现原则是：前端代码先迁移，接口契约仍以当前仓库后端为准；当前后端没有的接口先记录在本文档，不在本次迁移中直接实现。

## 1. 公告与推荐侧栏接口

建议后端新增位置：

- Controller：`backend/src/main/java/com/campusforum/announce/controller/AnnounceController.java`
- Service：`backend/src/main/java/com/campusforum/announce/service/AnnounceService.java`
- 如需持久化，可新增 announcement / recommendation 相关 domain、mapper、migration；也可以先基于帖子、话题、AI 推荐结果聚合生成。

公共说明：

- 前缀：`/api/v1/announce`
- 响应格式：沿用当前后端统一响应 `R<T>`，即 `{ code, message, data, traceId }`
- 前端用途：`Square.vue`、`Home.vue`、`PostDetail.vue`、`SidebarRight.vue`、`OfficialAnnouncementCarousel.vue`
- 当前前端已做降级：接口不可用时使用本地 fallback 数据或隐藏相关区域，不应影响页面主体渲染。

### GET `/api/v1/announce/daily-hot`

作用：获取每日热门文章，用于广场右侧热门文章列表，以及从热门文章入口跳转帖子详情。

Query：

| 参数 | 类型 | 必填 | 说明 |
|---|---|---:|---|
| `date` | string | 否 | 日期，格式建议 `yyyy-MM-dd`；不传则使用当天 |
| `limit` | number | 否 | 返回数量，建议默认 5，最大 20 |

Response `data`：

```json
{
  "items": [
    {
      "id": "hot-1",
      "articleId": 1,
      "postId": 1,
      "title": "热门帖子标题",
      "heatLabel": "12.5k 热度",
      "hasArticle": true,
      "summary": "内容摘要",
      "source": "校园广场",
      "publishedAt": "2026-06-22T10:00:00"
    }
  ],
  "generatedAt": "2026-06-22T10:00:00"
}
```

后端修改建议：优先在 `post` 模块已有帖子数据上按浏览、点赞、评论、收藏、时间衰减计算热度；若需要独立缓存，可增加热门结果缓存表或 Redis 缓存。

### POST `/api/v1/announce/daily-hot/clicks`

作用：记录热门文章点击，用于热度统计和推荐质量分析。

Body：

```json
{
  "articleId": "hot-1",
  "postId": 1,
  "source": "square-sidebar"
}
```

Response `data`：

```json
{ "accepted": true }
```

后端修改建议：在 `announce` 模块新增点击记录服务；可写入独立点击日志表，也可接入现有审计/统计体系。该接口不应返回敏感用户信息。

### GET `/api/v1/announce/official-announcements`

作用：获取官方公告轮播，用于首页和广场公告位。

Query：

| 参数 | 类型 | 必填 | 说明 |
|---|---|---:|---|
| `limit` | number | 否 | 返回数量，建议默认 3，最大 10 |
| `placement` | string | 否 | 展示位置，取值建议 `SQUARE` 或 `HOME` |

Response `data`：

```json
{
  "items": [
    {
      "id": "announcement-1",
      "title": "官方公告标题",
      "summary": "公告摘要",
      "description": "公告说明",
      "buttonText": "查看详情",
      "imageUrl": "https://example.com/banner.png",
      "badge": "官方公告",
      "meta": "教务通知",
      "link": "/posts/1"
    }
  ],
  "generatedAt": "2026-06-22T10:00:00"
}
```

后端修改建议：可先复用 `tenants.announcement` 提供单条租户公告；若要支持轮播、多位置和上下线管理，需要新增公告表，并在租户后台增加管理入口。

### GET `/api/v1/announce/ai-recommendations`

作用：获取 AI 推荐内容，用于右侧推荐栏。

Query：

| 参数 | 类型 | 必填 | 说明 |
|---|---|---:|---|
| `limit` | number | 否 | 返回数量，建议默认 5，最大 20 |

Response `data`：

```json
{
  "items": [
    {
      "id": "recommendation-1",
      "title": "推荐内容标题",
      "categoryLabel": "AI 推荐",
      "viewCount": 1200,
      "thumbnailUrl": "https://example.com/cover.png",
      "targetUrl": "/posts/1",
      "reason": "与你最近浏览的话题相关"
    }
  ],
  "generatedAt": "2026-06-22T10:00:00"
}
```

后端修改建议：建议在 `announce` service 中聚合 `post/search/ai` 模块能力；第一版可按热门帖子和用户关注话题生成，不必直接调用大模型。

### GET `/api/v1/announce/hot-topics`

作用：获取热门话题，用于趋势话题展示。

Query：

| 参数 | 类型 | 必填 | 说明 |
|---|---|---:|---|
| `limit` | number | 否 | 返回数量，建议默认 10，最大 30 |

Response `data`：

```json
{
  "items": [
    {
      "topic": "考研",
      "heat": 9820,
      "heatLabel": "9.8k",
      "targetUrl": "/search?keyword=考研"
    }
  ],
  "generatedAt": "2026-06-22T10:00:00"
}
```

后端修改建议：可从帖子 `topics/tags` 聚合统计，结合近 7 天活跃度排序；若已有搜索索引，也可从搜索热词或 MeiliSearch 统计数据生成。

### GET `/api/v1/announce/post-detail-sidebar`

作用：获取帖子详情页右侧作者摘要和推荐文章。

Query：

| 参数 | 类型 | 必填 | 说明 |
|---|---|---:|---|
| `postId` | number | 是 | 当前帖子 id |
| `authorId` | number | 是 | 当前帖子作者 id |
| `limit` | number | 否 | 推荐文章数量，建议默认 5，最大 10 |

Response `data`：

```json
{
  "author": {
    "authorId": 1,
    "nickname": "小青",
    "avatarUrl": "https://example.com/avatar.png",
    "bio": "个人简介",
    "label": "活跃作者",
    "postCount": 12,
    "followerCount": 30,
    "likeCount": 88
  },
  "recommendations": [
    {
      "id": "post-2",
      "postId": 2,
      "title": "相关帖子标题",
      "summary": "摘要",
      "viewCount": 100,
      "publishedAt": "2026-06-22T10:00:00",
      "targetUrl": "/posts/2"
    }
  ],
  "generatedAt": "2026-06-22T10:00:00"
}
```

后端修改建议：作者摘要可在 `user/post/follow` 模块聚合；推荐文章可复用 `post` 列表能力，按同作者、同话题、同空间优先排序。

## 2. 联调注意事项

- 新增接口未实现前，前端应保留降级数据，不应把 404 视为全局致命错误。
- 公告接口可以匿名访问，但点击记录如果绑定用户行为，后端应允许匿名记录或安全忽略用户身份。
- 不要在响应中返回私钥、API token、第三方凭据等敏感信息。
