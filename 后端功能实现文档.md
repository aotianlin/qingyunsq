# CampusForum 后端功能实现文档

本文档面向前后端联调和后续维护，说明当前后端的模块职责、核心流程、权限边界和主要副作用。

## 1. 技术架构

后端位于 `backend/`，技术栈：

- Java 17
- Spring Boot 3.3.0
- MyBatis-Plus 3.5.7
- MySQL 8
- Redis 7
- Sa-Token 1.38.0
- Caffeine 本地缓存
- WebSocket
- Knife4j / springdoc，本地开发可开启
- Apache Tika，用于上传文件真实 MIME 校验
- OWASP Java HTML Sanitizer，用于 UGC 内容净化
- Apache POI，用于后台 XLSX 导出
- LangChain4j / OpenAI compatible AI 调用

典型分层：

```text
controller -> service -> mapper -> domain
              |
              + dto / vo / infra / security / tenant
```

Controller 负责参数接收、登录用户获取和权限注解；Service 负责业务规则、事务、副作用；Mapper 使用 MyBatis-Plus。

## 2. 基础设施

### 2.1 统一响应与异常

所有普通 REST 接口使用 `R<T>` 包装：

- `code=0` 成功
- 失败由 `BusinessException` 或全局异常处理器转成统一响应
- 每个响应携带 `traceId`，优先来自 MDC

`GlobalExceptionHandler` 处理：

- Sa-Token 未登录：HTTP 401 + `UNAUTHORIZED`
- 无权限/角色：HTTP 403 + `FORBIDDEN`
- Bean Validation：HTTP 400 + 字段级错误信息
- 租户上下文缺失：HTTP 503 + `SERVICE_UNAVAILABLE`
- SSRF、MIME、加密等安全异常映射为对应错误码

文件下载、预览、导出接口直接写 `HttpServletResponse`，不一定使用统一响应。

### 2.2 鉴权

`SaTokenConfig` 对 `/api/v1/**` 生效：

- 放行登录、注册、验证码、重置密码、租户信息、下载/预览直链、AI 卡片批量缓存读取等接口
- GET 请求默认允许游客，但敏感 GET，如 `/auth/me`、通知、消息、关注状态等要求登录
- 非 GET 写操作全部要求登录

管理权限由 `AdminStpInterface` 提供：

- `TENANT_ADMIN` 拥有租户后台权限，如用户、帖子、空间、举报、敏感词、导出等
- `SUPER_ADMIN` 在租户权限基础上增加 `super:tenant:manage`
- 用户角色优先从 Sa-Token Session 读取，缺失时查库

### 2.3 租户隔离

租户上下文由 `TenantResolutionFilter` 在请求进入 MVC 前解析：

- `standalone`：固定租户 id，默认 `1`
- `multi`：已登录以 Session 中 `tenantId` 为权威；未登录按子域名或 `X-Tenant-Id` 解析

MyBatis-Plus 租户插件在 SQL 层自动追加 `tenant_id` 条件。租户无关表如 `tenants`、`achievements` 等按配置忽略。

`TenantBindingCheckInterceptor` 对已登录请求做额外防护：如果客户端携带 `X-Tenant-Id`，必须和后端解析出的租户一致，否则返回 `TENANT_VIOLATION` 并写审计。

### 2.4 限流

`WebMvcConfig` 注册两类限流：

- 通用 `RateLimitInterceptor`：作用于 `/api/v1/**`
- `AiRateLimitInterceptor`：作用于 `/api/v1/ai/**`，按用户每分钟和租户每天限流

配置位于 `application.yml`：

- 匿名默认 100 次/分钟
- 已登录默认 200 次/分钟
- 登录、注册、AI、上传、私信、下载/预览、后台导出等端点有单独配额

### 2.5 CORS 与 WebSocket

CORS 仅对 `/api/**` 生效，来源由 `security.cors.allowed-origins` 配置，`allowCredentials=false`。项目使用 `Authorization` 头，不依赖 Cookie。

WebSocket 地址为 `/ws/notify`，来源由 `ws.allowed-origins` 控制。推荐连接前调用 `/api/v1/auth/ws-ticket` 获取短期票据，避免主 token 出现在 URL、日志和 Referer 中。

## 3. 用户与认证模块

核心类：

- `AuthController`
- `UserController`
- `UserService`
- `EmailVerificationCodeService`
- `LoginLockoutService`
- `WechatMiniProgramClient`

### 3.1 注册

流程：

1. 邮箱归一化为小写
2. 检查邮箱和学号是否重复
3. BCrypt 加密密码
4. 根据学号前缀自动识别学院、专业、年级
5. 插入用户
6. 验证并消费注册验证码

验证码在插入成功后消费；如果插入并发冲突或验证码错误，事务回滚，验证码不会被吞掉。

### 3.2 登录

支持密码登录和邮箱验证码登录。

密码登录：

1. 解析可信代理后的真实 IP
2. 检查 IP 和账号锁定状态
3. 按租户和邮箱查询用户
4. 用户不存在或封禁时仍执行一次 BCrypt 校验，降低时序攻击风险
5. 失败累计账号和 IP 失败次数
6. 成功调用 `StpUtil.login`
7. Session 写入 `userId`、`role`、`tenantId`、`tenantCode`
8. 更新最后登录时间

验证码登录也复用锁定机制，失败返回不可区分错误。

### 3.3 密码变更

修改密码和重置密码成功后都会：

- 更新 BCrypt hash
- 清理 reset token
- 调用会话失效逻辑，踢掉该用户活跃 token
- 写审计和安全指标

前端必须主动清空本地 token 并跳转登录页。

### 3.4 用户资料

`GET /users/{id}` 只返回 `PublicUserVO`，避免公开接口泄露邮箱、学号、角色、状态等 PII。

更新头像和封面时：

- DTO 限制 URL 必须为 `http(s)`
- Service 校验域名白名单
- 上传头像/封面走 `StorageService`，再通过 `issuePublicGetUrl` 返回可访问 URL
- 上传图片使用 Tika 做真实 MIME 校验

## 4. 帖子模块

核心类：

- `PostController`
- `CommentController`
- `PostService`
- `CommentService`
- `FavoriteService`
- `MentionParser`
- `MarkdownEscaper`
- `PostViewDeduper`

### 4.1 发帖

主要流程：

1. 如果指定 `spaceId`，校验用户是该空间已加入成员
2. 写入作者、范围、空间、类型、标题、内容、话题、标签等字段
3. 正文经过 HTML Sanitizer 净化
4. 如传 `quotePostId`，生成引用内容并将类型设为 `QUOTE`
5. 默认状态为 `1`
6. 调用 AI/敏感词风控，风险高时将状态设为 `2`
7. 插入帖子
8. `type=QA` 时创建 `qa_questions` 扩展记录
9. 触发发帖成就
10. 写入 MeiliSearch 索引
11. 解析 `@昵称` 并发送提及通知
12. QA 标签命中用户订阅时发送标签订阅通知

### 4.2 帖子列表

`PostPageRequest` 支持：

- `scope`
- `authorId`
- `sort`
- `cursor`
- `cursorId`
- `limit`

普通列表只返回 `status=1`。`sort=following` 会查询当前用户关注列表并过滤作者；其他排序支持最新和热度。

### 4.3 帖子详情和浏览计数

详情读取时会调用 `viewPost`，使用 `PostViewDeduper` 按用户或 IP 在 30 分钟窗口内去重，避免刷新刷浏览量。

### 4.4 修改与删除

修改帖子要求作者本人。更新内容同样经过净化和风险检测，高风险会隐藏。

删除帖子要求作者本人；后台强删通过 `AdminPostController` 调用同一删除逻辑。

### 4.5 点赞与收藏

`reactions` 表用 `(user_id, target_type, target_id, type)` 唯一键表示一条反应。

切换逻辑：

- 已存在则删除并返回 `false`
- 不存在则插入并返回 `true`
- 帖子点赞会原子更新 `like_count`
- 新点赞会触发作者成就和点赞通知
- `COLLECT` 用于收藏列表，不增加 `like_count`

## 5. 评论模块

评论支持一级评论和扁平回复：

- `parentId`：父评论
- `replyToId`：回复目标评论

创建评论流程：

1. 敏感词风险等级检测，最高风险拒绝
2. 校验帖子存在
3. 校验父评论和回复目标归属关系
4. HTML 净化后写库
5. 触发评论成就
6. 原子增加帖子评论数
7. 通知帖子作者或被回复用户
8. 解析 @ 提及并通知
9. 给帖子作者 WebSocket 广播评论变更

评论列表先取根评论，再批量挂载子回复，避免每条评论单独查回复。

评论点赞仅支持 `LIKE`，会更新评论点赞数并通知评论作者。

## 6. 空间模块

核心类：

- `SpaceController`
- `SpaceService`
- `SpaceMapper`
- `SpaceMemberMapper`

### 6.1 空间创建

创建时校验：

- 分类必须在 `MAJOR/CLASS/CLUB/INTEREST`
- 可见性必须在 `PUBLIC/REVIEW/INVITE`

创建后：

- `spaces.member_count=1`
- 创建者自动写入 `space_members`，角色 `OWNER`，状态 `1`

### 6.2 加入与审核

加入规则：

- 已加入则报错
- 首次加入前检查用户加入空间数量上限
- `PUBLIC` 空间直接加入
- `REVIEW/INVITE` 空间写入待审核，并通知空间主

审批通过：

- 待审核记录状态改为 `1`
- 成员数原子 +1

移除成员：

- 不能移除 `OWNER`
- 状态改为 `3`
- 成员数原子 -1

退出空间：

- 空间主不能退出
- 成员状态改为 `2`
- 成员数原子 -1

### 6.3 空间访问控制

非 `PUBLIC` 空间读取帖子时，必须是已加入成员；否则返回无权限。历史实现中不存在 `PRIVATE` 可见性，当前逻辑统一为“非 PUBLIC 即需成员”。

`sensitiveWords` 属于管理配置，仅空间 `OWNER/ADMIN` 视图返回。

### 6.4 空间管理

空间 `OWNER/ADMIN` 可：

- 更新空间资料
- 审核或移除成员
- 查看全部帖子
- 管理空间帖子状态

租户管理员和超管可停用或解散空间。

## 7. 资源模块

核心类：

- `ResourceController`
- `ResourceService`
- `StorageService`
- `LocalStorageService`
- `OssStorageService`
- `SignedUrlService`
- `MimeTypeValidator`

### 7.1 上传

上传流程：

1. 检查文件名
2. 检查扩展名白名单
3. Tika 检测真实 MIME，并与扩展名交叉验证
4. 流式计算 SHA-256
5. 调用 `StorageService.upload(inputStream, filename, contentType, size)`
6. 用 SHA-256 查重，命中则删除新上传对象并复用已有资源
7. 写入资源元数据

后端默认允许：

```text
pdf, doc, docx, ppt, pptx, xls, xlsx, jpg, jpeg, png, gif, webp, md, markdown
```

默认不允许 zip、rar、7z。

### 7.2 可见性

访问资源时统一走 `ensureCanAccess`：

- 上传者本人可访问
- `TENANT_ADMIN/SUPER_ADMIN` 可访问
- `PUBLIC` 所有人可访问
- `SPACE` 仅关联空间成员可访问
- `PRIVATE` 仅上传者和管理员可访问
- 未知值按私有处理

无权访问时返回资源不存在，避免通过 id 枚举。

### 7.3 下载与预览

下载和预览支持签名 URL：

- 登录用户先申请 `/resources/{id}/signed-url`
- 浏览器用 `?sig=` 访问下载或预览
- 签名短期有效，避免主 token 暴露在 URL 中

下载会原子增加下载计数。

预览类型：

- PDF 和图片直接流式返回
- Markdown 直接返回文本 MIME
- Office 文档返回 JSON，前端使用 `previewServiceUrl` 和短期 `downloadPath`
- `preview-text` 支持 Markdown 和 DOCX 文本抽取

DOCX 文本抽取禁用 DOCTYPE 和外部实体，防 XXE。

## 8. 搜索模块

核心类：

- `SearchController`
- `SearchService`
- `MeiliSearchClient`
- `SearchSyncListener`

搜索支持帖子、用户、资源、空间四类。

帖子搜索策略：

1. 优先 MeiliSearch，并显式传入当前租户 id
2. 无结果或异常时回退 MySQL FULLTEXT
3. FULLTEXT 异常时回退 LIKE

用户搜索：

- 关键词长度小于 2 直接返回空
- 仅按昵称、学院、专业等公开信息检索
- 返回 `PublicUserVO`

资源和空间搜索使用 LIKE，并按资源 id 或空间成员数排序。

发帖、改帖、改状态、重建索引等操作会写入或更新搜索索引。

## 9. AI 模块

核心类：

- `AiController`
- `AiService`
- `MockAiService`
- `OpenAiCompatService`
- `TenantAwareAiService`
- `RagChatService`
- `PostAiCardService`
- `AiWorkspaceService`

### 9.1 AI 服务选择

基础配置在 `application.yml`：

- `ai.provider=mock|openai|ollama`
- `ai.base-url`
- `ai.api-key`
- `ai.model`

生产中租户 AI 配置优先级更高，存储在 `tenants.ai_config`，由 `TenantAwareAiService` 每次按当前租户解析。

租户 AI API Key 使用 AES-GCM 加密；旧 ECB 密文会兼容解密并异步升级。

### 9.2 RAG 问答

`RagChatService` 工作流：

1. 取最后一条 user 消息作为问题
2. 归一化并生成最多 5 个检索候选词
3. 按 `POST/RESOURCE/SPACE` 检索站内内容
4. 去重后最多保留 6 条引用
5. 构造上下文，要求模型优先依据用户上下文和站内资料回答
6. 调用 `AiService.chat`
7. 返回 `reply + citations`

`abilities` 不传或包含 `web-search` 时启用站内检索。

### 9.3 帖子智能卡片

缓存表：`post_ai_cards`

刷新条件：

- 缓存不存在
- 帖子 `updated_at` 比缓存版本新
- 评论数相对快照增加 10 条以上

生成输入包括标题、正文、类型、标签和最近高赞/新评论。AI 失败时不写脏数据；若有旧缓存则返回旧缓存，否则返回 `null`。

列表页应使用 `passive=true` 或批量缓存接口，避免一次列表触发大量 LLM 调用。

### 9.4 AI 工作台

`AiWorkspaceService` 是轻量原型实现：

- 数据持久化到 `backend/data/ai-workspace.json`
- 使用内存 `ConcurrentHashMap`
- 支持智能体、插件、知识库、文档导入任务、对话和反馈
- 部分公开种子数据 `ownerId=0`
- 用户相关操作通过 Sa-Token 获取 userId

该模块适合本地演示；生产化建议迁移到数据库表，并补充租户隔离、审计、配额、异步任务和文件存储。

## 10. 打卡模块

核心类：

- `CheckinController`
- `CheckinService`

### 10.1 挑战

挑战记录在 `checkin_challenges`：

- 创建者、空间、名称、描述、开始日期、结束日期、规则、成员数、状态
- 删除是将 `status` 设置为 0

详情会根据当前登录用户统计：

- 是否参与过
- 总打卡天数
- 当前连续天数

### 10.2 打卡

打卡流程：

1. 校验挑战存在且状态有效
2. 检查今天是否已经打卡
3. 判断是否首次参与
4. 写入记录，日期为服务器当天
5. 图片 URL 序列化为 JSON
6. 调用 AI 检查内容和挑战主题相关性，结果写入 `ai_check`
7. 首次参与时挑战成员数 +1
8. 触发打卡成就

数据库唯一键 `(challenge_id, user_id, checkin_date)` 兜底保证每天一次。

### 10.3 排行榜和分享

排行榜按用户聚合记录：

- 总天数降序
- 连续天数降序

连续天数要求最新日期是今天或昨天，否则连续数为 0。

分享打卡会生成一条广场帖子：

- `scope=SQUARE`
- `type=CHECKIN`
- 标题 `[打卡] 挑战名`
- 正文包含打卡内容和图片 Markdown

## 11. 问答模块

核心类：

- `QaController`
- `QaService`

QA 帖由发帖时 `type=QA` 自动创建 `qa_questions` 记录。

采纳回答规则：

1. 帖子必须存在
2. 帖子类型必须是 `QA`
3. 仅提问者可采纳
4. QA 记录必须存在
5. 已解决不能重复采纳
6. 评论必须存在且属于该帖子
7. 更新 `is_solved`、`accepted_comment_id`、`solved_at`
8. 如果被采纳者不是提问者，触发成就和采纳通知

## 12. 私信与通知

核心类：

- `MessageController`
- `MessageService`
- `NotifyController`
- `NotifyService`
- `SessionRegistry`
- `NotifyWebSocketHandler`

### 12.1 私信

私信发送规则：

- 不能发给自己
- 接收者必须存在
- 内容最大 2000 字
- 图片 URL 必须 `http(s)`
- 内容入库前使用 HTML Sanitizer
- 用敏感词服务评估风险等级，写入 `messages.ai_risk_level`
- 如果接收者在线，通过 WebSocket 发送 `MESSAGE` payload

对话列表通过 Mapper 查询每个会话最后一条消息，避免全量加载。

### 12.2 通知

通知创建规则：

- 不通知自己
- 接收者如果在 `mute_settings` 中屏蔽该类型，则不创建通知
- 写入 `notifications`
- WebSocket 尽力推送

通知类型包括：

- `COMMENT`
- `LIKE`
- `REPLY`
- `MENTION`
- `ACCEPT`
- `JOIN`
- `TAG_SUBSCRIBE`
- `MESSAGE`
- `SYSTEM`

## 13. 关注模块

核心类：

- `FollowController`
- `FollowService`

关注关系存储在 `follows`：

- 不能关注自己
- 重复关注幂等返回
- 取消关注直接删除匹配记录
- 粉丝和关注列表使用游标分页
- 计数用 count 查询

关注列表被帖子模块的 `sort=following` 使用。

## 14. 举报与敏感词

### 14.1 举报

`ReportService` 创建举报时读取当前登录用户作为 `reporterId`，状态默认为 `0`。

后台处理举报：

- 设置新状态
- 写入处理人
- 写入处理备注和处理时间
- 写审计日志

当前处理状态取值由前端约束，后端未做严格枚举校验，联调时建议只传 `0/1/2`。

### 14.2 敏感词

`SensitiveWordService` 支持普通词和正则词。

核心能力：

- 内容和词条统一做 NFKC、移除零宽字符、全角转半角、小写归一化
- 普通词用 contains
- 正则词预编译后用 find
- 按租户使用 Caffeine 缓存，TTL 60 秒，增删词主动失效
- 正则最长 200 字符，录入时校验合法性
- 返回命中的最高风险等级，0 表示未命中

敏感词检测用于：

- 发帖风控
- 评论风控
- 私信风险等级

## 15. 成就模块

核心类：

- `AchievementController`
- `AchievementService`

成就定义在 `achievements`，用户获得记录在 `user_achievements`。业务事件触发成就：

- 发帖
- 评论
- 点赞获得
- 打卡
- 回答被采纳

查询接口返回所有成就定义，并标注当前用户是否已获得及获得时间。

### 16.1 Dashboard

### 16.2 用户管理

用户管理支持：

- 列表筛选
- 改角色
- 封禁/解封
- 批量封禁/解封

防护：

- `ChangeRoleRequest` 仅允许 `USER/TENANT_ADMIN`
- 批量操作最多 100 个 id
- 角色变更、封禁、解封写审计
- 关键操作会踢出被影响用户会话

### 16.3 帖子与空间管理

管理端可：

- 查询包含隐藏状态的帖子/空间
- 切换置顶、精华
- 设置帖子或空间状态
- 强制删除帖子
- 解散空间

操作均写入审计日志。

### 16.4 导出

导出拆分为四个细粒度权限：

- `tenant:export:users`
- `tenant:export:posts`
- `tenant:export:audit`
- `tenant:export:reports`

支持 CSV 和 XLSX。用户导出默认脱敏；`fullPii=true` 仅 `SUPER_ADMIN` 可用。导出接口按端点限流 1/min。

### 16.5 租户管理

仅 `SUPER_ADMIN` 可管理租户。

租户停用流程：

1. 切换 `tenants.status`
2. 立即失效 `ActiveTenantCache`
3. 停用时枚举该租户活跃用户并 `StpUtil.kickout`

AI 配置管理：

- API Key 加密存储
- GET 不回传明文
- PUT 支持 partial update
- 掩码 key 原样回传视为不修改
- `provider=openai` 时校验 baseUrl 不能指向内网
- 更新后失效 `TenantAwareAiService` 客户端缓存
- 写租户审计，apiKey 只记录掩码差异

## 17. 文件存储

`StorageService` 抽象：

- `upload(InputStream, filename, contentType, size)`
- `download(storageKey)`
- `delete(storageKey)`
- `issuePublicGetUrl(storageKey)`

实现：

- `LocalStorageService`：本地开发
- `OssStorageService`：阿里云 OSS

安全要点：

- 已移除 `/uploads/**` 静态目录映射
- 资源访问必须走 Controller 可见性校验或签名 URL
- 上传必须传真实 `file.getSize()`，避免流读取截断

## 19. 联调风险点

- `GET /users/{id}` 返回公开用户，不是完整用户
- 前端私信 `receiverId` 建议传数字
- 前端资源上传 accept 包含压缩包，但后端默认不允许
- Office 预览返回 JSON 描述，不是文件流
- 下载、预览、导出需按 blob/流处理
- 已登录后前端不要携带 `X-Tenant-Id`
- 密码变更和重置成功后前端要主动登出
- AI 工作台当前是 JSON 文件持久化，不是生产级数据模型
- 后台部分 Map 入参如举报处理、帖子/空间状态设置需要前端保证字段存在和类型正确
